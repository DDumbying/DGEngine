# DGEngine

DGEngine (Dumbest-Game-Engine) is a small game engine written in C for learning how game engines work internally.

The project focuses on simplicity, readability, and low-level understanding rather than building a full-featured production engine.(i am not sure actually, who knows..)

The main goal of DGEngine is to explore:
- Rendering pipelines
- Game loops
- Resource management
- Engine architecture
- OpenGL rendering
- Basic game systems
- Data organization and memory handling

## Philosophy

DGEngine is intentionally minimal.

This project is not trying to compete with large engines like Unity or Unreal Engine.
Instead, it is designed to:
- stay understandable
- remain hackable
- avoid unnecessary abstractions
- teach low-level concepts through practice

## Current Goals

- Linux-only support
- Simple OpenGL renderer
- SDL2 platform layer
- Basic texture rendering
- Camera system
- Input handling
- Simple entity system
- Basic collision handling
- Small playable game built using the engine
